* 功能
 - 加载Java类的字节代码到java虚拟机中
 - 虚拟机默认行为足够满足大多数情况下的需求
 - 如果需要与类加载器进行交互,则需要修改类加载器
* 基本概念
* java虚拟机使用java类的方式
* java虚拟机与程序的生命周期
** 终止的情况
 - 执行System.exit()
 - 执行结束
 - 执行中遇到异常或错误
 - 操作系统错误
* 类的加载连接和初始化
** 加载
 - 查找并加载类的二进制数据
** 连接
 - 验证
  + 确保加载类的正确性
 - 准备
  + 为类的静态变量分配内存,并设置为默认值
 - 解析
  + 将类中的符号引用转换为直接引用
** 初始化
 - 为类的静态变量赋予正确的初始值
* java程序对类的使用
** 方式
 - 主动使用
  + 创建类的实例 new
  + 静态方法或属性的操作
  + 反射
  + 初始化类的一个子类
  + java虚拟机启动时标记为启动的类
 - 被动使用
** 特点
 - xc所有java虚拟机实现必须在每个类或接口被java程序首次主动使用时才初始化他们
